# Kadr — полное руководство по функциям видеоредактора

> Kadr — GPU-ускоренный многодорожечный видеоредактор (Electron + React +
> TypeScript) со встроенными ИИ-ассистентами Codex и Claude Code, локальным
> распознаванием речи и программируемой анимацией на Remotion.
> Превью компонуется на GPU (WebGL2), экспорт строго WYSIWYG: тот же
> компоновщик кадр в кадр рендерит итоговый файл.

## 1. Быстрый старт

### Запуск

```bash
npm install          # один раз (postinstall сам пересоберёт node-pty под Electron)
npm run dev          # запуск в режиме разработки
npm run build        # production-сборка в out/
npm run typecheck    # проверка типов (рендерер + main-процесс)
```

### Минимальный рабочий цикл

1. **Импорт** — кнопка «Импорт» в панели «Медиа» (или перетащите файл из
   панели прямо на таймлайн).
2. **Монтаж** — расставьте клипы на дорожках, подрежьте края, добавьте
   переходы и эффекты.
3. **Просмотр** — Space, перемотка кликом по линейке, покадрово ←/→.
4. **Экспорт** — кнопка «Экспорт», выбор пресета, готово.

### Что умеет редактор — кратко

- Многодорожечный монтаж: видео, аудио, текст, Remotion-фрагменты.
- GPU-превью в реальном времени, идентичное финальному рендеру.
- Ключевые кадры на всех параметрах трансформации и масок.
- 3D-повороты слоёв с честной перспективой.
- 14 переходов наложения + 12 кинематографичных торцевых переходов.
- Эффект «Внешнее сияние» — неравномерный дым с частицами.
- Локальное распознавание речи (faster-whisper large-v3) с пословными
  таймкодами и защитой от галлюцинаций.
- Авто-субтитры с караоке-анимацией одним диалогом.
- Remotion-фрагменты: программируемая анимация как клип на таймлайне,
  превью без рендера, hot reload.
- Встроенные Codex и Claude Code с MCP-доступом ко всему проекту.
- Экспорт: аппаратное кодирование WebCodecs, motion blur, frame blending,
  пресеты под YouTube/Shorts/WebM/MP3.
- Автосохранение раз в 5 минут.

## 2. Интерфейс

Окно редактора состоит из пяти зон:

```
┌─────────────────────────────────────────────────────────────┐
│  Топбар: Kadr · имя проекта · Отменить/Повторить ·          │
│  Новый/Открыть/Сохранить/Сохранить как… · Экспорт · 🤖 · RU │
├───────────────┬──────────────────────────┬──────────────────┤
│               │                          │                  │
│  Боковая      │   Превью (GL-канвас)     │   Инспектор      │
│  панель       │   + индикатор звука      │   («Свойства»)   │
│  («Медиа»)    │   + транспорт            │                  │
│               │                          │                  │
├───────────────┴──────────────────────────┴──────────────────┤
│  Таймлайн: тулбар · линейка · дорожки с клипами             │
└─────────────────────────────────────────────────────────────┘
```

### Топбар

| Кнопка | Действие |
|---|---|
| ↶ / ↷ | Отмена / повтор (в подсказке — что именно отменится) |
| Новый проект | Чистый проект (несохранённые изменения теряются) |
| Открыть проект | Диалог открытия `.kadr` |
| Сохранить | Ctrl+S — в текущий файл (или диалог, если не сохранялся) |
| Сохранить как… | Ctrl+Shift+S — всегда диалог; проект «переезжает» по новому пути |
| Экспорт | Диалог рендера |
| Codex/Claude + 🤖 | Выбор агентской среды и открытие/закрытие её сессии |
| RU/EN | Переключение языка интерфейса |

### Зоны можно растягивать

- Вертикальный разделитель между боковой панелью и превью — ширина панели.
- Горизонтальный разделитель над таймлайном — высота таймлайна.
- Размеры запоминаются между запусками (localStorage).

### Превью

- Отображает проект в реальном времени через WebGL2-компоновщик — то, что
  вы видите, и есть будущий рендер (один и тот же код рисования).
- Поверх канваса живут оверлеи Remotion-фрагментов и гизмо перемещения.
- Справа — индикатор уровня звука.
- Снизу — транспорт: undo/redo, в начало, play/pause, таймкод.

### Инспектор («Свойства»)

Без выделения — настройки проекта: имя, разрешение (W/H), частота кадров.

С выделенным клипом:
- имя клипа;
- для текстовых клипов — текст, размер шрифта, цвет, обводка, жирность;
- позиция X/Y, масштаб, поворот, прозрачность (числами);
- громкость и «Без звука»;
- секция **Эффекты** с пресетами (⭐).

## 3. Проекты

### Формат

Проект — один JSON-файл с расширением `.kadr`. Внутри — полная модель:
дорожки, клипы, ссылки на медиафайлы (по абсолютным путям), ключевые
кадры, эффекты, тексты, фрагменты. Медиафайлы в проект не копируются.

### Сохранение

- **Ctrl+S / «Сохранить»** — в текущий файл; если проект ещё не имеет
  файла, откроется диалог.
- **Ctrl+Shift+S / «Сохранить как…»** — всегда диалог; после сохранения
  проект продолжает жить по новому пути (последующие Ctrl+S — туда).
- Все диалоги (проекты, импорт, экспорт) запоминают последнюю папку;
  первый запуск открывается в «Видео»/«Загрузки».

### Автосохранение

Работает само, настройки не требует:

- **раз в 5 минут**, и только если проект менялся с прошлого раза;
- файл `<имя>.autosave.kadr` **в той же папке, что и основной** (для ещё
  не сохранённых проектов — в «Загрузках»), перезаписывается;
- запись **атомарная** (временный файл + переименование) — сбой в момент
  записи не может оставить битый автосейв;
- **пропускается**, пока идёт экспорт или открыта сессия встроенного
  ИИ-агент (чтобы не конкурировать за диск и не снимать проект посреди
  массовых правок); после завершения — запишется при следующем тике;
- пустые проекты не сохраняются.

Восстановление: «Открыть проект» → выбрать `.autosave.kadr`.

### Отмена/повтор

- История на 50 шагов, каждый шаг подписан («перемещение клипа»,
  «разрезание», «выбор перехода»…) — подпись видна в подсказке кнопок
  ↶/↷.
- Ctrl+Z — отмена, Ctrl+Shift+Z или Ctrl+Y — повтор.
- Откатывается всё: правки клипов, ключевые кадры, эффекты, переходы,
  тексты, применение пресетов.

## 4. Импорт медиа

### Способы импорта

- Кнопка **«Импорт»** в панели «Медиа» (мультивыбор).
- Файлы анализируются ffprobe: длительность, размеры, fps, наличие
  аудио, миниатюры (начало и конец для видео), волновая форма звука.

### Поддерживаемые форматы

- **Видео**: mp4, mkv, mov, webm, avi, m4v, mts.
- **Аудио**: mp3, wav, flac, ogg, aac, m4a, opus.
- **Изображения**: png, jpg/jpeg, webp, bmp, gif.
- **Текст**: srt, txt — попадают в секцию «Тексты» (см. §20).

### Панель «Медиа»

- Сетка карточек: миниатюра (для аудио — ♪), имя, полный путь в
  подсказке.
- **Перетаскивание** карточки на таймлайн — вставка клипа в указанное
  место; **двойной клик** — вставка на плейхед.
- Видео со звуком вставляется **парой связанных клипов**: видео на
  видеодорожку + звук на аудиодорожку (двигаются вместе, см. §7).
- Бейджи на карточке:
  - `⚙ 47%` — строится прокси;
  - `P` — прокси готов, превью идёт через него;
  - `📝` — кнопка транскрибации (для медиа со звуком).

### Секция «Тексты»

Сворачиваемый список под сеткой медиа (стрелка ▾/▸ в заголовке, счётчик
документов, собственная прокрутка — длинный список не вытесняет медиа).
Подробности — в §20.

## 5. Прокси

Тяжёлые исходники тормозили бы превью, поэтому:

- каждое видео с коротком стороной **≥720p** автоматически получает
  фоновую облегчённую копию: H.264 540p + AAC 96k;
- прокси строятся **по одному** (очередь), прогресс виден на карточке;
- кэш: `userData/proxies/`, ключ — путь+размер+mtime файла (изменился
  исходник — прокси перестроится);
- превью декодирует прокси, **экспорт всегда читает оригинал** — качество
  рендера не страдает.

## 6. Таймлайн и дорожки

### Тулбар таймлайна

| Кнопка | Действие |
|---|---|
| +Видео | Добавить видеодорожку (сверху) |
| +Аудио | Добавить аудиодорожку (снизу) |
| 📝 Транскрибировать | Распознать речь в выделенном диапазоне (§19) |
| ✨ Авто субтитры | Анимированные субтитры на диапазон/весь проект (§21) |
| Высота | Слайдер высоты дорожек (32–140 px) |
| Масштаб | Слайдер зума таймлайна (логарифмический) |

### Дорожки

- Порядок видеодорожек = порядок слоёв: **верхняя дорожка рисуется поверх
  всех**.
- У каждой дорожки:
  - **имя** (V1, V2…, A1…);
  - **🔇 mute** — видеодорожка исчезает из кадра, аудио замолкает;
  - **🔒 lock** — клипы не редактируются, вставка на дорожку запрещена;
  - **слайдер**: для аудио — громкость дорожки (0–200 %), для видео —
    **непрозрачность всего слоя** (0–100 %);
  - **🌐 движение дорожки** (видео) — анимированная 3D-трансформация
    целиком всей дорожки (общий «полёт камеры» по слою): X/Y/масштаб/
    поворот/наклоны/глубина с ключевыми кадрами в абсолютном времени
    проекта;
  - контекстное меню — удалить дорожку, добавить дорожку рядом.

### Навигация

- **Клик по линейке** — переставить плейхед (с перетаскиванием).
- **Колесо мыши** — зум вокруг курсора.
- **Shift+перетаскивание по линейке** — выделить **диапазон** (фрагмент):
  жёлтая зона. Диапазон используется экспортом, транскрипцией,
  авто-субтитрами, копированием/удалением фрагмента.
- **Esc** — снять диапазон.
- **Ctrl+клик по пустому месту дорожки** — сомкнуть клипы (закрыть дыру
  слева от места клика).

### Курсор-плейхед

Красная вертикаль через все дорожки. Во время воспроизведения движется по
«якорным» часам — равномерно, без накопления дрожания таймера.

## 7. Клипы: базовые операции

### Виды клипов

| Вид | Цвет | Содержимое |
|---|---|---|
| media (видео) | синий | кадры видеофайла/изображения |
| media (аудио) | зелёный | звук |
| text | особый | текстовый оверлей |
| remotion | фиолетовый ⚛ | живая Remotion-композиция (§22) |

### Операции мышью

- **Перетаскивание** — перемещение по дорожке и между дорожками
  (с примагничиванием к соседним клипам и плейхеду).
- **Края клипа** — подрезка начала/конца (трим). У начала меняется
  `inPoint` (откуда играть исходник).
- **Верхние уголки** — ручки **фейдов**: потяните внутрь — плавное
  появление/затухание (видео — прозрачность, звук — громкость).
- **Перетаскивание края с зажатым Ctrl** — изменение **скорости** клипа
  (растяжение/сжатие по времени с пересчётом темпа).
- **Нижние фиолетовые уголки** — «кончики» торцевых переходов (§16).
- **Двойной клик** — открыть редактор анимации (§9).
- **Контекстное меню** — разрезать, удалить, переходы, связка и т. д.

### Растяжение за длину исходника — луп

Если видео/аудио-клип растянут дольше, чем длится исходник, источник
**зацикливается** (играет по кругу). Удобно для фоновой музыки и текстур.

### Скорость

`speed` ≠ 1 ускоряет/замедляет клип; звук обрабатывается цепочкой
`atempo` в финальном миксе (без изменения высоты тона). Время исходника
для кадра вычисляется как `inPoint + rel·speed`.

### Связанные клипы (видео+звук)

Видео со звуком вставляется парой с общим `linkId`:
- двигаются и подрезаются вместе;
- **U** — разорвать/восстановить связь выделенного;
- видеоклип пары автоматически замьючен (звук идёт с аудиодорожки).

### Разрезание и удаление

- **S** — разрезать все выделенные клипы (или клип под плейхедом) в
  позиции плейхеда.
- **D / Delete / Backspace** — удалить выделенные клипы; если выделения
  нет, но есть диапазон — **вырезать диапазон** из всех дорожек со
  смыканием.
- **Ctrl+C / Ctrl+V** — копирование клипов (или содержимого диапазона) и
  вставка на плейхед.

## 9. Трансформации и ключевые кадры

### Параметры трансформации клипа

Каждый видимый клип имеет:

| Параметр | Диапазон | Описание |
|---|---|---|
| X, Y | px проекта | смещение от центра кадра |
| Масштаб | 0.01–20 | относительно вписанного размера |
| Поворот | градусы | вокруг центра слоя (ось Z) |
| Прозрачность | 0–1 | умножается на фейды и непрозрачность дорожки |
| Наклон X/Y | градусы | 3D-наклоны (§10) |
| Глубина Z | px | 3D-смещение от камеры (§10) |

Числами параметры правятся в Инспекторе, анимация — в редакторе анимации.

### Редактор анимации (AnimEditor)

Открывается **двойным кликом по клипу**. Режимы: **Трансформация** и
**Маска** (вкладки).

- Каждый параметр — строка с числовым полем и кнопкой-ромбом ◇/◆:
  - ◇ — параметр статичен; клик ставит **ключевой кадр** в позиции
    плейхеда;
  - ◆ — на плейхеде есть ключ; клик убирает его.
- Ключи отображаются маркерами на мини-таймлайне клипа; перетаскиваются.
- Между ключами значения **интерполируются** (линейно / сглаженно —
  выбор easing на ключе).
- Изменение значения при включённой анимации записывает ключ в текущей
  позиции — рабочий процесс как в AE: двигаете плейхед → крутите ручку →
  ключ ставится сам.
- Время ключей хранится в секундах **от начала клипа** — клип можно
  свободно двигать по таймлайну, анимация едет с ним.

### Что анимируется

- Вся трансформация (X, Y, масштаб, поворот, прозрачность, наклоны, Z).
- Громкость клипа (gain).
- Маска: все четыре кромки и каждый параметр каждой фигуры (§11).
- Движение дорожки (🌐) — в абсолютном времени проекта.

## 10. 3D-режим

- Включается в редакторе анимации (переключатель 3D) — у клипа появляются
  **наклон X**, **наклон Y** и **глубина Z**.
- Перспектива честная: углы слоя проецируются с гомогенной координатой,
  текстура интерполируется перспективно-корректно (без «излома» по
  диагонали).
- Фокусное расстояние «камеры» — 1.2 максимальной стороны кадра.
- Работает на любых клипах, включая Remotion-фрагменты (для них превью
  автоматически переключается в режим захвата пикселей, §22).
- Движение дорожки (🌐) — те же 3D-параметры, применяемые ко всем клипам
  дорожки **после** их собственных трансформаций: общий облёт сцены.

## 11. Маски

Два механизма, работают одновременно:

### Кромочная маска (crop)

Четыре анимируемые кромки — слева/сверху/справа/снизу, каждая 0–50 %
слоя. Обрезка прямоугольником, вычисляется в шейдере.

### Фигурные маски

До **8 фигур** на клип, каждая:

| Свойство | Описание |
|---|---|
| Тип | прямоугольник / эллипс / треугольник |
| Центр (cx, cy) | в координатах слоя (0–1), анимируемый |
| Размер (w, h) | анимируемый |
| Растушёвка внутрь | мягкая кромка внутрь от границы |
| Растушёвка наружу | мягкая кромка наружу |
| Инверсия | фигура **вырезает дыру** вместо «оставить внутри» |

Логика комбинирования: **объединение** обычных фигур **минус** все
инвертированные. Расстояния до кромок считаются аспектно-корректно
(растушёвка одинаково широка по всем направлениям).

Редактирование — в редакторе анимации, режим «Маска»: фигуры
добавляются, перетаскиваются и растягиваются мышью прямо в превью.

## 12. Пресеты поз

Сохранение **одного состояния** маски или трансформации под именем — и
применение в любом проекте, в любое время, к любому клипу.

- Кнопка **⭐ Пресеты** в редакторе анимации.
- Меню показывает пресеты текущего режима (трансформация/маска).
- **Сохранить**: ввести имя → текущие значения параметров (или набор
  фигур маски) записываются.
- **Применить**: клик по имени. Значения пишутся через обычный механизм

  применяется попараметрно (анимируемо), с другим — заменяет набор фигур.

- Хранение: `pose-presets.json` в данных приложения — пресеты живут
  между перезапусками и общие для всех проектов.

## 13. Эффекты

Эффекты живут в секции **Эффекты** Инспектора выделенного клипа. У
каждого эффекта: галочка включения, кнопка удаления ✕, свои параметры.
Эффекты работают на любых клипах — видео, изображениях, тексте,
Remotion-фрагментах — и в превью, и в рендере одинаково.

### Внешнее сияние (Outer Glow)

Не плоский ореол, а **неравномерный дым с частицами**, хаотично
выходящий из-за краёв объекта:

| Параметр | Диапазон | Что делает |
|---|---|---|
| Цвет | пипетка | базовый цвет свечения |
| Размер | 4–400 px | радиус ореола |
| Яркость | 0–3 | сила свечения (выше ~1.5 — начинает «выгорать») |
| Насыщенность | 0–2 | 0 — монохром, 2 — кислотный перенасыщенный |
| Дымность | 0–1 | 0 — чистый ровный ореол, 1 — полный хаос клубов |
| Скорость дыма | 0–3 | темп дрейфа клубов и мерцания частиц |
| Частицы | 0–1 | плотность круглых «угольков», уносящихся от кромки |

Как это устроено (и почему выглядит дорого):

- слой рендерится в отдельный буфер; его альфа-силуэт размывается
  спиральным фильтром в поле ореола (¼ разрешения — быстро);
- шум FBM с **поворотом октав** (никаких «линейных волн») и
  billow-профилем даёт круглые пухлые клубы, разделённые тёмными
  трещинами;
- низкочастотная огибающая по периметру делает свечение неравномерным:
  где-то еле тлеет, где-то вырывается далеко за радиус;
- отдельный слой **хлопьев** отрывается от внешней кромки и уплывает;
- сама граница ореола «гуляет» по шуму — рваные щупальца вместо ровного
  круга;
- частицы — круглые мерцающие точки из клеточного хеша, каждая со своим
  размером и фазой;
- свечение подавлено внутри объекта и мягко гаснет на внешней границе;
  яркость проходит через «киношное плечо» — пересветы сохраняют текстуру.

Анимация дыма привязана к локальному времени клипа: в превью и в
экспорте дым **идентичен** кадр в кадр. Эффект корректно работает внутри
переходов и при motion blur.

## 15. Переходы наложения

Vegas-стиль: **перекройте два клипа на одной видеодорожке** — зона
перекрытия автоматически становится переходом.

- На зоне перекрытия рисуется заметная полоса; клик по значку —
  **меню выбора эффекта** (15 пунктов, включая «Без перехода»).
- Оба клипа рендерятся в отдельные буферы и смешиваются GLSL-шейдером
  (семейство gl-transitions) — любые маски/эффекты/анимации внутри
  клипов продолжают работать.
- Длительность перехода = длительность перекрытия (двигайте клипы).
- **Звук** перекрывающихся клипов автоматически кроссфейдится — и в
  превью, и в экспортном миксе.
- «Без перехода (склейка)» — жёсткий кат в точке начала второго клипа.

Список переходов — Приложение А. Работает между любыми видами клипов,
включая Remotion-фрагменты.

## 16. Торцевые переходы

AE/BorisFX-стиль: эффектные переходы **на кончиках клипов**, без
перекрытия — для смежных («впритык») клипов и одиночных входов/выходов.

### Как добавить

- **Фиолетовые уголки** в нижних углах каждого клипа: клик по левому —
  In-переход (вход), по правому — Out-переход (выход). В меню — эффект и
  длительность.
- **Значок ◈ на стыке** двух смежных клипов: ставит **парный** переход
  (Out первого + In второго того же типа) — кат прячется в пике эффекта,
  два кончика читаются как одно непрерывное движение.
- Полоски на кончиках клипа показывают установленные переходы.

### Характер

Эффекты сделаны «дорого»: 14–22-точечные размытия со **спектральным
расщеплением** (хроматическая аберрация вдоль движения), квартическая
кривая с максимумом скорости в точке стыка (длинный замах → резкий
рывок), параболические траектории у вип-панов, виньетка.

12 эффектов: зум-блюр (наезд/отъезд), RGB-расхождение, вип-паны в четыре
стороны, вращение с размытием, засветка-вспышка, глитч, упругое
растяжение, линзовое искажение. Полный список — Приложение А.

## 18. Аудио

### Громкость

- **Клип**: параметр «Громкость» (0–200 %), анимируемый ключами (gain —
  это Anim, как X/Y); «Без звука» — галочка.
- **Дорожка**: слайдер громкости 0–200 %.
- Итог = громкость клипа × громкость дорожки × фейды.

### Фейды и кроссфейды

- Ручки в верхних углах клипа — fade in/out (звук и картинка вместе).
- Перекрытие клипов на одной дорожке — автоматический аудиокроссфейд.

### Превью-звук

Аудио маршрутизируется через WebAudio: громкость >100 % реально усиливает
сигнал; индикатор уровня — справа от превью. Элементы синхронизируются с
мастер-часами (пересинхронизация при дрейфе >0.15 с).

### Экспортный микс

Финальный звук собирает ffmpeg по сегментам:
`громкость → atempo (скорость) → afade (фейды) → adelay (позиция) → apad
→ amix`. Все потоки выровнены по длительности, масштабирование amix
компенсируется точно — громкости в миксе соответствуют превью. Лупы
клипов разворачиваются в несколько сегментов, скорость — цепочкой atempo.

### Волновые формы

На аудиоклипах рисуется двухслойная волна (пик + RMS) в стиле Audacity,
до 1000 точек/сек.

## 19. Распознавание речи

Локальная транскрипция на **faster-whisper** — без интернета и подписок.

### Запуск

- **📝 на карточке медиа** (панель «Медиа») — распознать **весь файл**;
  таймкоды реплик — от начала исходника.
- **📝 Транскрибировать** в тулбаре таймлайна — распознать **выделенный
  диапазон**: микшируется всё слышимое в диапазоне (ровно как в
  экспорте — с громкостями, скоростями, фейдами) и распознаётся.

### Диалог транскрипции

| Поле | Варианты |
|---|---|
| Модель | **large-v3** (максимум качества, ≈2× длительности записи), medium (в 2–4 раза быстрее), base (черновик) |
| Язык | автоопределение / русский / английский |
| Разбивка | **1 слово (караоке)** / 2 / 3 / 4 слова / целые фразы |
| Таймкоды (для диапазона) | **абсолютные** (по таймлайну) / относительные (от начала фрагмента) |

Во время работы — прогресс и **живой текст** распознаваемых реплик;
можно отменить.

### Качество и защита от галлюцинаций

Whisper склонен «слышать» текст в тишине и музыке. Включены все обороны:

- **VAD** (детектор речи) вырезает неречевые участки до распознавания;
- отключена «память» между сегментами — главный источник зацикленных
  повторов;
- пороги уверенности (log-prob, no-speech, compression ratio);
- пропуск текста, «услышанного» внутри длинных пауз;
- пост-фильтры: сегменты с равномерно низкой пословной уверенностью и
  бесконечные повторы одной фразы отбрасываются.

Контракт закреплён тестом: чистый тон обязан дать **ноль** реплик.
На реальной русской речи пословная уверенность ≈1.0.

### Разбивка по словам

Whisper отдаёт время **каждого слова**. Реплики нарезаются с правилами:

- не больше N слов (1–4) на реплику;
- реплика не перепрыгивает конец предложения и паузы >0.6 с;
- слова распределяются **ровно**: 7 слов при лимите 3 → 3+2+2 (никаких
  болтающихся одиноких слов);
- составные слова («мини-программами») не рвутся: токены-продолжения
  склеиваются обратно.

### Результат

`имя.srt` + `имя.txt` рядом с медиафайлом (для диапазона — рядом с
доминирующим исходником, имя вида `проект_0m12s-0m45s.srt`). Файлы не
перезаписывают чужие: при коллизии добавляется суффикс. Оба документа
регистрируются в секции «Тексты».

## 20. Панель субтитров

### Для SRT

- список реплик: номер, таймкоды начала/конца (правятся как текст в
  формате `чч:мм:сс,мс`), текст (многострочный);

### Для TXT

Простое редактирование всего текста.

### Внешние правки

Панель раз в 2 секунды проверяет mtime файла: если документ изменил
кто-то снаружи — например, **встроенный агент отредактировал SRT своими

несохранённых правок).

## 21. Авто-субтитры

Кнопка **«✨ Авто субтитры»** в тулбаре таймлайна: речь → стильные
анимированные субтитры одним диалогом.

### Настройки

| Поле | Варианты |
|---|---|
| Цель | выделенный диапазон, иначе весь проект |
| Модель / язык | как в транскрипции |
| Слов в титре | 1 (караоке) / 2 / 3 / 4 |
| Шрифт | Sans / Arial Black / Serif / Mono |
| Размер | стартовый размер в px (дальше — мышью) |
| Жирный | да/нет |
| Цвет / Цвет выделения | пипетки |
| Появление | **Поп (пружина)** / Подъём / Фейд / Нет |
| Выделение слова | **Караоке-цвет** / Подпрыгивание / Плашка / Нет |
| Скорость анимации | 0.5×–2× |

### Что происходит

1. Распознаётся речь цели (с пословными таймкодами; SRT попутно
   сохраняется в «Тексты»).
2. Генерируется **прозрачный Remotion-фрагмент** с запечёнными репликами
   и стилем — каждое слово анимируется ровно в момент произнесения:
   - *караоке-цвет* — произнесённые слова перекрашиваются и остаются
     подсвеченными до конца реплики;
   - *подпрыгивание* — активное слово упруго увеличивается;
   - *плашка* — активное слово на цветной плашке;
   - появление реплики — пружинный поп / подъём снизу / фейд.
3. Клип субтитров ложится на свободную верхнюю дорожку над целью, в
   нижней трети кадра, и сразу выделен.

### Мышь: позиция и размер

У выделенного фрагмента в превью — пунктирная рамка (гизмо):

- **перетаскивание рамки** — позиция;
- **уголок** или **колесо мыши** — масштаб;
- каждый жест — шаг истории (Ctrl+Z работает).

Гизмо работает для **любых** Remotion-фрагментов, не только субтитров.

### Дальнейшие правки

Субтитры — обычный фрагмент: код открыт (`index.tsx` в workspace),
правится руками или встроенным агентом («сделай плашку скруглённой и
жёлтую обводку»), превью обновляется горячо, рендер — один, при экспорте.

## 22. Remotion-фрагменты

Программируемая анимация (React/TSX) как полноценный клип на таймлайне.

### Концепция

- **Фрагмент** = композиция Remotion в общем workspace
  `~/kadr-fragments` (один node_modules на все проекты, ~150 МБ,
  ставится автоматически при первом использовании).
- На таймлайне — фиолетовый клип ⚛: двигается, режется, миксуется
  переходами, принимает эффекты и маски, как любой клип.
- Бывают **прозрачными** (титры/анимации поверх нижних слоёв — по
  умолчанию) и **непрозрачными** (самодостаточные сцены с фонами и
  звуком). Звук композиции слышен в превью и попадает в экспортный микс.
- Создаются с fps = max(60, fps проекта) в размере проекта.

### Превью без рендера — гибрид двух режимов

1. **iframe-оверлей** (по умолчанию): страница с @remotion/player
   позиционируется поверх GL-канваса по трансформации клипа,
   синхронизирована с часами редактора (seek/play/pause/громкость).
   Нулевые накладные расходы; оверлей обрезается границами кадра превью.
   Dev-сервер vite даёт **hot reload**: правка кода композиции видна в
   превью через ~1–2 секунды. Никаких рендеров при итерациях.
2. **Захват пикселей** (автоматически, когда на фрагменте есть
   GL-возможности — эффекты, 3D, маски, движение дорожки, любые
   переходы, перекрытия): невидимое offscreen-окно гонит кадры
   композиции (≤1280 px по ширине) прямо в компоновщик — фрагмент
   становится обычным GL-слоем, и **всё** (3D-перспектива, маски,
   переходы, сияние, порядок дорожек) работает живьём. Убрали
   эффект/3D — захват сам выключается, возвращается лёгкий iframe.

### Экспорт — один рендер

При экспорте каждый фрагмент рендерится `remotion render`-ом **один
раз** в полном качестве (прозрачные — VP8 с альфа-каналом, непрозрачные —
H.264) и подставляется как видеослой; дальше работает обычный
WYSIWYG-конвейер. Рендеры кэшируются **по хешу содержимого** папки
фрагмента: повторный экспорт без правок берёт готовый файл мгновенно.

### Структура workspace

```
~/kadr-fragments/
  package.json            remotion, @remotion/player, react, vite
  player.tsx              страница плеера (управляется редактором)
  src/Root.tsx            регистрация композиций для remotion render
  src/fragments/
    index.ts              авто-генерируемый реестр
    <имя>-<id>/
      index.tsx           ← код композиции (правьте свободно)
      meta.json           размер/fps/длительность/прозрачность
```

Правила: модуль экспортирует `fragment = { component, meta }`;
длительность в `meta.json` держите в согласии с клипом; медиа для
композиции кладите в папку фрагмента и импортируйте
(`import bg from './bg.jpg'`) — абсолютные пути не переживут финальную
сборку рендера.

### Создание

- через встроенного Codex или Claude (основной путь, §23–24);
- программно: `kadrEditor.createFragment({ name, start, end,
  transparent })`;
- авто-субтитры создают фрагмент сами (§21).

Клип ложится на верхнюю свободную видеодорожку (или создаётся новая).

## 23. Встроенные Codex и Claude Code

Селектор **Codex/Claude** и кнопка 🤖 в топбаре открывают терминал (xterm)
с настоящим интерактивным CLI, **подключённым к открытому проекту**.
По умолчанию выбран Codex; выбор сохраняется между запусками. Смена среды
при открытой панели завершает текущую сессию и сразу запускает выбранную.

### Сессия

- запускается ваш CLI `codex` или `claude` в PTY — полный интерактивный
  TUI, как в обычном терминале; используется существующий вход CLI;
- дополнительное окружение (прокси и т. п.) задаётся в
  `~/.config/kadr/codex-env.json` или `claude-env.json` →
  `{ "env": { "HTTPS_PROXY": "...", … } }` и пробрасывается в сессию;
- первоначальную авторизацию или подтверждение доверия выполняйте прямо
  в открытом TUI;
- рабочая папка — папка `.kadr`-файла (или домашняя);
- повторное нажатие 🤖 или ✕ — сессия и мост убиваются;
- переопределение запуска в соответствующем `*-env.json` —
  `{ "command": "...", "args": [...], "env": {...} }`; `args` полностью
  заменяет автоматически созданные аргументы интеграции;
- Codex получает MCP через одноразовые `-c mcp_servers.kadr...` параметры,
  не меняющие `~/.codex/config.toml`; модель, sandbox и approvals остаются
  пользовательскими. Если sandbox запрещает прямую запись файла фрагмента,
  агент использует `readTextFile` / `writeTextFile` через `kadr_eval`.

### Что агент видит и умеет

Через MCP-сервер `kadr` (подключается автоматически, **не** заменяя ваши
собственные MCP-серверы):

- читает весь проект: дорожки, клипы, тайминги, пути всех медиафайлов,
  транскрипты;
- выполняет любые операции редактора (с undo-метками);
- запускает экспорт и транскрипцию, создаёт Remotion-фрагменты;
- плюс обычные инструменты выбранного CLI: файлы, shell (ffmpeg/ffprobe
  доступны), ваши другие MCP.

### Примеры команд

- «Что сейчас на таймлайне?»
- «Отрендери все куски аудио в mp3, где нет видео, и положи в отдельную
  папку.»
- «Сделай на выделенном диапазоне анимированную заставку с названием
  канала.» — создаст фрагмент и будет править его код по вашим правкам,
  изменения видны в превью через секунды.
- «Перенарежь субтитры по 2 слова и сделай их жёлтыми.»
- «Поставь кроссфейды по 0.5 с между всеми клипами на V1.»

## 24. MCP-инструменты

Полный список инструментов сервера `kadr`:

### kadr_state

Снимок живого состояния: проект целиком (дорожки → клипы со всеми
параметрами; ассеты с абсолютными путями; texts — транскрипты с путями к
файлам), путь проекта, выделение, плейхед, доступные пресеты экспорта.

### kadr_eval

Выполнить JavaScript в странице редактора (тело async-функции,
`return` — результат). Доступно всё API `window.kadrEditor` (§28): store
с действиями (`pushHistory`, `updateClip`, `insertClipFromAsset`,
`splitAtPlayhead`, `setTransition`…), `probeMedia`, чтение/запись
проектов. Документация по API встроена в описание инструмента.

### kadr_export

Отрендерить проект (или диапазон) и дождаться завершения: outputPath,
presetId, start/end, motionBlur, frameBlending. Тот же WYSIWYG-конвейер,
что и кнопка «Экспорт».

### kadr_transcribe

Распознать речь: либо весь медиафайл (`assetId`), либо диапазон
таймлайна (`start`+`end`); модель, язык, режим таймкодов, слов в
реплике. Пишет SRT+TXT рядом с исходником, регистрирует в проекте,
возвращает пути и превью реплик. Файлы агент может читать и править
напрямую — панель субтитров подхватит.

### kadr_fragment_create

Создать Remotion-фрагмент на `[start, end)`: имя, прозрачность.
Возвращает id, путь к TSX и URL живого плеера. Дальше агент правит файл
обычными инструментами — превью обновляется горячо; рендер один, при
экспорте.

| WebM VP9 | WebM | VP9 (перекодирование) | |
| MP3 | MP3 | только аудио | рендер видео пропускается |

### Диапазон

Если на таймлайне выделен диапазон (Shift+драг по линейке) — рендерится
**только он**; без диапазона — весь проект.

### Опции

- **Размытие движения** (по умолчанию вкл.) — §26.
- **Frame Blending** (по умолчанию вкл.) — §26.

### Как идёт рендер (фазы прогресса)

1. **Рендер Remotion-фрагментов** — если есть; один раз на фрагмент,
   с кэшем по содержимому.
2. **Рендер видео** — офлайн-проход тем же GL-компоновщиком, что и
   превью: кадры кодируются WebCodecs (предпочтительно аппаратно VAAPI,
   иначе программно), пишутся в временный MP4.
3. **Аудио и мастеринг** — ffmpeg микширует звук по сегментам и
   муксует/перекодирует в финальный контейнер по пресету.

Экспорт можно отменить на любой фазе. Диалог сохранения помнит последнюю
папку экспорта.

### Скорость

Быстрый путь декодирования: mp4box-демультиплексация + WebCodecs-декодер
на клип, последовательное чтение с упреждением — **~8× быстрее**
покадровых перемоток. На неподдерживаемых файлах (битый mp4, экзотический
кодек, webm) — автоматический откат на классический путь: экспорт может
замедлиться, но никогда не сломается и не заморозится.

### Гарантия WYSIWYG

Превью и экспорт — один код рисования: одинаковые трансформации, маски,
эффекты, переходы, цветовые матрицы (untagged-потоки приводятся к той же
эвристике, что у превью: HD→BT.709, SD→BT.601). Что видите — то и
получите.

## 26. Качество движения

### Размытие движения (Motion Blur)

Галочка в диалоге экспорта (по умолчанию включена):

- 180°-затвор: каждый кадр — среднее **8 подкадров**, распределённых по
  половине кадрового интервала;
- между подкадрами движутся **трансформации, маски, движение дорожки,
  переходы** — быстрые анимации получают киношный смаз вместо
  стробоскопа;
- декодирование не дорожает: подкадры переиспользуют кадры видео.

### Frame Blending

Галочка в диалоге экспорта (по умолчанию включена):

- проблема: исходник 25 fps в проекте 60 fps повторяет кадры неравномерно
  (2-3-2-3) — движение внутри клипа пульсирует;
- решение: соседние кадры источника смешиваются пропорционально точной
  фазе времени — движение контента становится равномерным; помогает и
  замедленным клипам;
- включается **только там, где нужно** (скорость·fps исходника < fps
  проекта): совпадающие и более быстрые источники не трогаются вообще —
  проверено бит-в-бит.

### Равномерность превью

Часы воспроизведения «якорные»: время проецируется от точки старта по
vsync-меткам кадров, дрожание таймера не накапливается — движение в
превью равномерное.

## 27. Производительность

- **Прокси 540p** для тяжёлых исходников в превью (§5).
- **Пауза почти бесплатна**: без изменений превью перерисовывается ~4
  раза в секунду вместо 60 (загрузка текстур видео не бесплатна).
- **GL-текстуры кэшируются** по клипам; неиспользуемые выгружаются.
- **Захват фрагментов** ограничен 1280 px по ширине и включается только
  при необходимости (§22).
- **Экспорт** не трогает аудио-двойники видеоклипов (звук идёт из
  ffmpeg-микса) и декодирует каждый кадр ровно один раз.
- Рендеры фрагментов и прокси строятся **очередями** по одному — слабые
  CPU не захлёбываются.
- Аппаратный энкодер (VAAPI) используется автоматически, если Chromium
  его поддерживает; иначе — программный.

`window.kadrEditor` — публичная поверхность для скриптов, тестов и MCP:

| `useSettings` | язык, высота дорожек |
| `usePosePresets` / `useFxPresets` | пресеты поз и эффектов |
| `PRESETS` | пресеты экспорта |

Ключевые действия store (`useEditor.getState()`):

- `pushHistory(labelKey)` — **один раз перед** дискретной правкой
  низкоуровневыми сеттерами (высокоуровневые — `addTrack`,
  `insertClipFromAsset`, `splitAtPlayhead`, `deleteSelection`,
  `addTexts`, `insertFragmentClip` — кладут историю сами);
- `updateClip(id, patch)`, `setClipDuration`, `setClipSpeed`;
- `insertClipFromAsset(assetId, trackId|null, atSec)`, `addAsset`;
- `addTrack(kind)`, `addTrackNear(trackId)`;
- `select(ids)`, `setPlayhead(t)`, `setRange({start,end}|null)`;
- `splitAtPlayhead()`, `deleteSelection()`, `copySelection()`,
  `pasteAtPlayhead()`;
- `setTransition(clipId, type|null)` — переход наложения;
- `setEdgeTransitions(...)` — торцевые переходы;
- `setProject(project, path?)` — загрузить проект целиком.

`window.kadr` — IPC-мост к main-процессу: `probeMedia`, `readProject` /
`writeProject` / `autosaveProject`, `readTextFile` / `writeTextFile` /
`statFile`, `transcribe`, `fragment*`, `agent*` (и совместимые `claude*`), экспортные вызовы.

Все времена везде — **секунды**. `tracks[0]` — верхняя видеодорожка.

## 30. Устранение неполадок

**Превью дёргается на тяжёлом видео** — дождитесь постройки прокси
(бейдж ⚙ на карточке), оно включится само.

**Готовый файл «дёргается» при просмотре, а превью плавное** — почти
наверняка ваш плеер не вытягивает программный декод 1080p60. Проверьте
файл на другом устройстве или плеером с аппаратным декодом
(`mpv --hwdec=vaapi`). Сам файл равномерный — тайминги и движение
проверены инструментально.

**В рендере «нет» эффекта/перехода** — проверьте, **тот ли файл** вы
открыли: типичный случай — просмотр старого рендера из другой папки.
Диалог экспорта показывает, куда пишет.

**Whisper придумал текст в тишине/музыке** — включены все фильтры, но на
экзотике возможно: перегенерируйте с явным языком вместо «авто» или
моделью large-v3; правьте реплики в панели субтитров.

**Фрагмент показывает «unknown composition»** — клип ссылается на
удалённую композицию: удалите клип или восстановите папку фрагмента в
`~/kadr-fragments/src/fragments/`.

**Codex/Claude не запускается в панели** — проверьте, что выбранный CLI
работает в обычном терминале; нестандартный путь/окружение задаётся в
соответствующем `~/.config/kadr/codex-env.json` или `claude-env.json`.

**Экспорт упал на конкретном файле** — экспорт сам откатывается на
медленный надёжный путь декодирования; если файл совсем битый —
переконвертируйте его (`ffmpeg -i in -c copy out.mp4`).

**Порт 5621 занят** — это dev-сервер фрагментов; живой сервер от
предыдущего запуска переиспользуется автоматически, осиротевший —
убивается сторожем. Вручную: `pkill -f kadr-fragments`.

## 32. Приложение Б

### Модель проекта (упрощённо)

```ts
Project {
  name, width, height, fps, background
  tracks: Track[]          // tracks[0] — верхняя видеодорожка
  assets: MediaAsset[]     // импортированные файлы (пути, метаданные)
  texts:  TextDoc[]        // транскрипты/субтитры (ссылки на файлы)
}

Track {
  kind: 'video' | 'audio'
  name, muted, locked, gain      // gain видео = непрозрачность слоя
  motion?: Transform3D           // 🌐 движение всей дорожки (анимируемое)
  clips: Clip[]
}

Clip {
  kind: 'media' | 'text' | 'remotion'
  assetId? | text+textStyle? | fragmentId+fragmentMeta?
  start, duration, inPoint, speed    // секунды; duration сверх исходника = луп
  fadeIn?, fadeOut?
  gain: Anim, muted
  transform: { x, y, scale, rotation, opacity, rotX?, rotY?, z? }  // все Anim
  mask?: { left, top, right, bottom }            // кромки, Anim
  maskShapes?: MaskShape[]                       // до 8 фигур
  effects: Effect[]                              // glow и будущие
  transitionIn?, transitionOut?: { type, duration }  // duration 0 = наложение
  linkId?                                        // связка видео+аудио
}

Anim { value: number, keyframes?: { time, value, easing }[] }
```

### Соглашения

- Все времена — **секунды**; времена ключей — от начала клипа; движение
  дорожки — в абсолютном времени проекта.
- `transitionIn` с `duration > 0` — торцевой переход; с `duration = 0` —
  тип смешивания для зоны наложения.
- Эффект: `{ id, type: 'glow', enabled, params }`.
